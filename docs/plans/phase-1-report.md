# Phase 1 Report — Prune Inapplicable Tools

**Date:** 2026-04-18
**Status:** ✅ Complete
**Plan reference:** [excel-specific-tooling-plan.md](excel-specific-tooling-plan.md)

---

## Outcome

The MCP server's exposed tool surface is now aligned with what makes sense against a single Excel-add-in WebView2 taskpane. Browser-centric tools that assumed the user could open new tabs, navigate URLs, install Chrome extensions, or emulate arbitrary devices have been removed.

## Tool-surface changes (user-visible)

| Tool                           | Before   | After    | Notes                                                                                              |
| ------------------------------ | -------- | -------- | -------------------------------------------------------------------------------------------------- |
| `install_extension`            | present  | **gone** | Chrome extensions don't exist in the WebView2 add-in host.                                         |
| `uninstall_extension`          | present  | **gone** | Same.                                                                                              |
| `list_extensions`              | present  | **gone** | Same.                                                                                              |
| `get_extension_permissions`    | present  | **gone** | Same.                                                                                              |
| `reload_extension`             | present  | **gone** | Same.                                                                                              |
| `new_page`                     | present  | **gone** | Add-in host owns its URL/window lifecycle.                                                         |
| `navigate_page`                | present  | **gone** | Same.                                                                                              |
| `resize_page`                  | present  | **gone** | Add-in host owns pane dimensions.                                                                  |

- `skills/excel-webview2/SKILL.md` — the two notes that told users not to call `navigate_page` / `new_page` now say the tools no longer exist and that re-attaching via `list_pages` + `select_page` is the right response to add-in-driven navigation.

## Internal plumbing retained on purpose

The plan offered to delete `src/utils/ExtensionRegistry.ts` _if nothing else imports it_. In this repo, `McpContext`, `McpResponse`, and `ToolDefinition` all import it, so the conditional is false. Keeping the registry is a deliberate scope-control decision — untangling it belongs in a follow-up cleanup, not Phase 1.

Similarly, `args.categoryExtensions` is read in three places (`McpResponse.ts`, `pages.ts` description, `script.ts` service-worker branch). Those reads are now unreachable (the CLI flag is gone and always `undefined`), but the branches themselves were left in — removing them would require touching `script.ts`'s service-worker routing, which is out of Phase 1 scope.
